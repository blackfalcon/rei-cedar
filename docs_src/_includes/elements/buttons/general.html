Sizes

Fancy smaller buttons?

Add .btn-sm or .btn-xs for additional sizes.

Note: inline form and input-groups use the .btn-sm class by default.

Block-level

Create block level buttons—those that span the full width of a parent— by adding .btn-block.

Making groupings of buttons flow correctly

A singleton button possesses no margin

Buttons which have a previous sibling which is also a button receive a left margin of 8px, this accounts for the fact that buttons are display: inline-block which adds an additional 4px of space totaling 12px.

If your buttons will wrap prior to displaying block you will have an undesirable staircase. To rectify this place buttons in a container with the btn-group inline-group classes.

Links acting as buttons

If the <a> elements are used to act as buttons – triggering in-page functionality, rather than navigating to another document or section within the current page – they should also be given an appropriate role="button".

Cross-browser rendering

As a best practice, we highly recommend using the <button> element whenever possible to ensure matching cross-browser rendering.

Among other things, there's a bug in Firefox <30 that prevents us from setting the line-height of <input>-based buttons, causing them to not exactly match the height of other buttons on Firefox.
